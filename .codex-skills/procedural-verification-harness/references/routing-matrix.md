# Routing Matrix

Use the smallest discipline that matches the task. Combine disciplines only
when the task genuinely crosses categories.

## Signals

| Signal | Apply |
| --- | --- |
| bug, crash, error, traceback, failing, not working | Investigation protocol |
| game, Godot, web, HTML, SVG, canvas, chart, UI, render, animation | Verification grounding |
| multi-step, new project, feature batch, long task, finish all, take it all the way | Multi-step loop |
| review, audit, regression risk | Review protocol plus investigation for plausible findings |
| ambiguous target, unknown project area | Context gate before decomposition |
| same blocker repeated, open-ended quality ceiling | Capability ceiling report or escalation |

## Rules

- Do not load every reference for every task.
- Do not apply the persistent goal ledger to small one-turn edits unless it
  improves handoff or evidence tracking.
- Do not turn the upstream Claude hook behavior into automatic file or setting
  edits. In Codex, routing is instruction-driven unless a user asks to install
  actual hooks.
